Scope:

This application will enable the faculty of the CSE Dept to make entries of syllabus completion and track the same.

Constraints and Assumptions:

All users are faculty of the CSE Dept with one/multiple admins. The applications will be accessible via the campus intranet (moodle??).

Reqmnt ID

Requirement Definiton
RID 1

Faculty must be able to upload an excel file containing their plan for a particular subject. This in turn must generate a series of checkboxes which can be checked upon completion.

RID 2

The completion status must be entered within 2 days, after which it can be unlocked only by admin

RID 3

Dashboard: with buttons for courses, announcements, settings such as changing userid and password. If possible, make customizable
RID 4

Attendance and internal assessment marks of classes handled must be made available.
RID 5

Statistics such as percentage of completion, lead/lag from original plan, days left for completion before tests/VTU exams, performance graphs of students

RID 6

Application must grant admin privileges. Access to all faculty completion statuses, plans and students’ performances

Security:

User data needs to be kept secure. All data is to be ultimately moved to a server within the college.

Communication Interfaces:

The system is available through the campus network.

Interface:

The application is web based

Error Handling:

User is directed to an appropriate webpage or issued a message through a dialog box
